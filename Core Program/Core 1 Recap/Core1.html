from sklearn.datasets import fetch_openml
from sklearn.model_selection import train_test_split

# Load the Fashion-MNIST dataset
fashion_mnist = fetch_openml(name='Fashion-MNIST')

# Split the data into training and test sets
X_train, X_test, y_train, y_test = train_test_split(
    fashion_mnist.data, fashion_mnist.target, test_size=0.2, random_state=42)

# Print the shapes of the resulting arrays
print(f"Training data shape: {X_train.shape}, training labels shape: {y_train.shape}")
print(f"Test data shape: {X_test.shape}, test labels shape: {y_test.shape}")

from sklearn.ensemble import RandomForestClassifier
import time

# Train a Random Forest classifier
start_time = time.time()

rfc = RandomForestClassifier(n_estimators=100, random_state=42)
rfc.fit(X_train, y_train)

elapsed_time = time.time() - start_time
print(f"Training time: {elapsed_time:.2f} seconds")

# Evaluate the model on the test set

3. Next, use PCA to reduce the dataset's dimensionality¶

In [5]:
from sklearn.decomposition import PCA

# Create a PCA object with an explained variance ratio of 95%
pca = PCA(n_components=0.95, random_state=42)

# Fit the PCA object to the training data and transform both the training and test data
X_train_pca = pca.fit_transform(X_train)
X_test_pca = pca.transform(X_test)

# Print the new shapes of the transformed arrays
print(f"Training data shape after PCA: {X_train_pca.shape}")
print(f"Test data shape after PCA: {X_test_pca.shape}")

Training data shape after PCA: (56000, 187)
Test data shape after PCA: (14000, 187)

The features of the dataset were still 784 before the PCA. After the pca this was reduced to 187

5. Was training much faster?¶

Yes, training the Random Forest classifier on the original dataset took 66.76 seconds, while training on the reduced dataset took 110.99 seconds. So training on the original dataset was actually faster in this case.

It's possible that the additional time is being spent on the PCA transformation step. However, the reduction in the number of features from 784 to 187 should still lead to faster training times for some machine learning algorithms.

9. What can you conclude from the results?¶

From the results, we can see that the original random forest classifier achieved a test accuracy of 0.88, while the reduced dataset random forest classifier achieved a slightly lower test accuracy of 0.86. This indicates that while PCA can be a useful technique for reducing the dimensionality of a dataset, it may not always result in improved model performance.

Additionally, we can see that the softmax regression classifier achieved a test accuracy of 0.85, which is lower than the accuracies achieved by the random forest classifiers. This suggests that the random forest classifiers may be a better choice for this particular dataset and classification task.

Finally, we can also see that the training times for the random forest classifiers were significantly longer than the training time for the softmax regression classifier, especially for the reduced dataset. This suggests that if training time is an important consideration, the softmax regression classifier may be a better choice.
